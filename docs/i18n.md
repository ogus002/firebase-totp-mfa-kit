# i18n

Phase 1 ships English-only. Locale override is via props.

## Override texts

```tsx
import { en, type MfaTexts } from '@/components/totp-mfa/lib/i18n';
import { TotpEnroll } from '@/components/totp-mfa/components/TotpEnroll';

const ko: Partial<MfaTexts['enroll']> = {
  title: '2단계 인증 설정',
  description: 'Authenticator 앱으로 QR 코드를 스캔하고 6자리 코드를 입력하세요.',
  submit: '확인 및 활성화',
  codePlaceholder: '123456',
};

<TotpEnroll user={user} issuer="Acme" texts={ko} />
```

## Full override

If you want to override every string, copy `en` from `lib/i18n.ts` and translate:

```ts
const ko: MfaTexts = {
  enroll: { /* ... */ },
  signIn: { /* ... */ },
  recovery: { /* ... */ },
  common: { /* ... */ },
};
```

## Locale router integration

The kit doesn't bind to `next-intl`, `react-i18next`, or any specific library.
Pass strings in directly:

```tsx
import { useTranslations } from 'next-intl';

function Page() {
  const t = useTranslations('mfa.enroll');
  return (
    <TotpEnroll
      user={user}
      issuer="Acme"
      texts={{
        title: t('title'),
        description: t('description'),
        // ...
      }}
    />
  );
}
```

## Error code mapping

The kit returns Firebase error codes (`auth/invalid-verification-code`, etc.).
The `mfa-errors.ts` source has English defaults. To translate:

```ts
const errorMessages: Record<string, string> = {
  'auth/invalid-verification-code': '코드가 올바르지 않습니다. 다시 시도하세요.',
  'auth/requires-recent-login': '다시 로그인해주세요.',
  // ...
};

// In your error handler:
const msg = errorMessages[error.code] ?? errorMessages.default;
```
